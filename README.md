<h2>Tech X Hunt 2026</h2>
<h3>Registration Platform</h3>

<p>
  This platform was developed for <strong>KLE Tech CodeClub</strong> to manage registrations for the 
  <strong>Flagship Tech X Hunt 2026</strong> event.<br>
  As a Core Member of CodeClub, the objective was to design and deploy a 
  <strong>reliable</strong>, real-world system capable of handling <strong>live users</strong>.
</p>

<h3>Tech Stack</h3>

<h4>Frontend</h4>
<ul>
  <li>React (Vite) with TypeScript</li>
  <li>Tailwind CSS for styling</li>
  <li>Framer Motion for animations and transitions</li>
  <li>Lucide React for icons</li>
</ul>

<h4>Backend</h4>
<ul>
  <li>FastAPI (Python) with asynchronous request handling</li>
</ul>

<h4>Database</h4>
<ul>
  <li>MongoDB Atlas using Motor (async driver)</li>
</ul>

<h4>Hosting</h4>
<ul>
  <li>Backend: Render</li>
  <li>Frontend: Render</li>
</ul>

<h3>Features</h3>
<ul>
  <li>Inspired by One Piece, the interface follows a cinematic, event-driven design with responsive layouts and animated elements.</li>
  <li>Team-based registration capturing captain and member details in a guided flow.</li>
  <li>Hard limit of 107 teams enforced at the backend level with concurrency-safe handling.</li>
</ul>

<h3>Scale and Usage</h3>
<ul>
  <li>Successfully handled <strong>107 team registrations</strong></li>
  <li>Deployed and used during a live event</li>
  <li>Maintained stable operation during active usage</li>
</ul>

<h3>Challenges and Improvements</h3>
<ul>
  <li>Managed backend cold start delays on free-tier hosting</li>
  <li>Refined UI consistency and communication elements post-deployment</li>
  <li>Identified the need for stricter validation, including duplicate entries and input constraints</li>
</ul>

<h3>Conclusion</h3>
<p>
  This project served as a practical <strong>full-stack deployment</strong> experience, involving 
  <strong>live debugging</strong> and <strong>user interaction management</strong>.<br>
  It provided valuable insights into building and operating systems under real-world conditions, 
  highlighting areas for improving <strong>reliability</strong> and <strong>scalability</strong> in future implementations.
</p>
